About Demos

Demos is a blockchain built for agent commerce — where autonomous AI agents can identify each other, coordinate, exchange value, and verify outcomes with cryptographic proof.

What is Demos?

Most blockchains today are designed for people or applications. Demos is different. It is designed for autonomous agents — AI systems that can act on behalf of users, services, or organizations across different networks.

As agents become more capable, they need reliable ways to:

  • Prove who they are across different platforms and networks
  • Check claims, credentials, and evidence from counterparties
  • Negotiate and settle transactions automatically
  • Create records that can be audited later

Demos provides protocol infrastructure to support these interactions.

What Demos is all about

Demos focuses on making agent-to-agent commerce practical and verifiable. Instead of humans manually managing every step, agents should be able to handle structured economic activity on their own.

To support this, Demos emphasizes four key properties:

  • Identity — Agents can carry a consistent identity across chains, platforms, and traditional systems.
  • Verification — Agents can check claims, credentials, and supporting evidence.
  • Atomicity — Transactions can settle cleanly without unnecessary intermediate exposure.
  • Audit — Important steps can leave a clear, verifiable record.

These properties support a standard five-stage flow for agent commerce: Identify → Vet → Negotiate → Settle → Verify.

Why agent commerce matters

Agent commerce refers to autonomous systems conducting economic activity — discovering counterparties, negotiating terms, exchanging value, and verifying outcomes — with minimal human intervention.

For this to work reliably at scale, agents need shared infrastructure that supports:

  • Portable identity — So an agent can prove who it is across different platforms and networks.
  • Verifiable claims — So counterparties can check credentials, permissions, or reputation signals.
  • Atomic settlement — So value can move cleanly without unnecessary counterparty risk.
  • Auditability — So important steps leave a clear, reviewable record.

Without these foundations, agent-to-agent interactions remain fragmented, manual, or limited to low-stakes use cases.

Cross-Context Identity

One important concept in Demos is Cross-Context Identity, or CCI. CCI allows agents and users to carry identity across different Web2 and Web3 contexts, instead of appearing as a separate actor in every system.

This matters because agent commerce depends on knowing who is acting, what authority they have, what evidence supports their claims, and whether the interaction can be reviewed later.

Where DNO fits in

This is where DNO, the Demos Network Oracle, fits in.

DNO is an independent, watch-only observer of public Demos infrastructure. It does not operate the Demos protocol, control validators, or certify identities, counterparties, or transactions.

DNO observes public validators and selected public infrastructure endpoints, applies a published methodology, and publishes structured assessments about what it can see.

DNO as an observability layer

DNO functions as an independent observability layer on top of the Demos public testnet. It continuously monitors public validators and selected public infrastructure endpoints, then reduces those observations into structured, machine-readable assessments.

Its primary output is the /organism feed, which reports:

  • Current observed network status, trend, and risk
  • Data quality and confidence in the assessment
  • Whether visible public nodes appear aligned
  • Active public incidents (if any)
  • Explicit signals when data is insufficient or uncertain

DNO does not control validators, participate in consensus, or influence protocol behavior. It only observes public signals and publishes what it can see, along with clear indications of what it cannot determine.

This separation allows agents, developers, and researchers to consume network context without relying solely on statements from the protocol team.

The /organism feed

DNO publishes its core public network assessment through a machine-readable feed called /organism.

/organism is DNO’s primary public assessment output. It returns a structured snapshot of what DNO currently observes about the public Demos network, including status, data quality, agreement, risk, incidents, and uncertainty.

The feed is designed for both humans and machines. Developers, dashboards, and agents can read it directly.

/organism is public-network only. DNO’s reference fleet and diagnostic layers may be shown elsewhere for transparency, but they do not determine the core public assessment.

For agents and developers

Use GET /organism as DNO’s primary public state feed. Read status, risk, and confidence for the minimal operating picture. Add data_quality, staleness_seconds, and agreement_reason before making automated interpretations.

Validate integrations against /organism/schema. Treat unknown, insufficient, and uncertain as valid states.

DNO informs context. It does not advise, predict, score, certify, verify identities, or decide action.

Honest uncertainty

DNO does not force incomplete observations into confident answers.

States like unknown, insufficient, and uncertain are valid outputs. They mean the visible public data is incomplete, stale, or conflicting — not that the API has failed.

This is an important part of DNO’s role. When DNO cannot see enough, it should say so clearly.

What DNO does

  • Observes public validators on the Demos testnet
  • Reports whether visible public nodes appear reachable and aligned
  • Publishes structured assessments that humans and agents can read
  • Records notable observed public network conditions over time
  • Shows when available data is insufficient or uncertain

What DNO does not do

To remain independent, DNO deliberately avoids roles that would turn observation into control or certification.

  • It does not operate the Demos protocol
  • It does not control, rank, or recommend validators
  • It does not issue, verify, or certify CCI or agent identities
  • It does not verify individual commerce claims
  • It does not approve or reject transactions
  • It does not provide financial, trading, or transaction advice
  • It does not predict future network behavior

In short: DNO observes and reports. It informs context; it does not advise, predict, score, certify, verify identities, or decide action.

Demos provides infrastructure for agents to identify, coordinate, settle, and verify across networks.

DNO’s role is narrower: make the observable public state around the Demos testnet easier to inspect.
